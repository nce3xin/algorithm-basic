# 连续子数组的最大和

这其实是一个动态规划的题目。这题目比较特殊，可以用一个变量s来做动态规划。

s存的是：以 前一个数结尾的子数组的和的最大值。

分情况讨论：

- 当s > 0 时，s+x
- 当 s < 0 时，x。因为前面最大都会小于0，所以一定不会接上前面。前面不管怎么接，对我们来说都是一个负收益。那么以当前数结尾的子数组的和的最大值就是x。
- 当 s == 0 时，x。这种情况可以和上一种情况合二为一。

注意，代码里写的时候和想的时候是有些出入的。下面这种写法更简洁。

```
if s <= 0 {
    s = 0
}
s = s + x
```
